## 1. Prerequisites

- **Jetson Orin Nano 8GB Developer Kit**
- **JetPack 6.0** or later (Ubuntu 22.04, Python 3.10)
- **128GB SD card** (formatted and flashed with JetPack image)
- 2Г— A4TECH 1080p USB Webcams
- Tower Pro Servo Motor (connected to GPIO pin 32)
- 7-inch LCD display (HDMI or DSI)
- 12V LED strip (always-on power supply)

## 2. Initial Jetson Setup

```bash
# Update system packages
sudo apt update && sudo apt upgrade -y

# Install system-level dependencies
sudo apt install -y \
    python3-pip python3-dev \
    libopencv-dev python3-opencv \
    libglib2.0-0 libsm6 libxext6 libxrender-dev \
```

## 3. Conda Environment Setup


## 5. Install PyTorch for Jetson (JetPack 6.x)

> **Important:** Do NOT use `pip install torch` вЂ” it installs the x86 version.
> Use NVIDIA's official Jetson wheel instead.

```bash
# Install the Jetson-optimised PyTorch wheel
pip install --no-cache-dir \
    https://developer.download.nvidia.com/compute/redist/jp/v60/pytorch/torch-2.3.0a0+ebedce2.nv24.04.14643755-cp310-cp310-linux_aarch64.whl

# Verify GPU is accessible
python -c "import torch; print(torch.cuda.is_available())"
# Expected output: True
```

## 6. Install Project Dependencies

```bash
conda activate chick_env
pip install --break-system-packages -r requirements.txt

# If opencv-python conflicts with system opencv, use headless build:
# pip install opencv-python-headless --break-system-packages
```

---

## 7. Install Jetson.GPIO

```bash
# Jetson.GPIO is included in JetPack but needs pip-level linking
pip install Jetson.GPIO --break-system-packages

# Add your user to the GPIO group (allows non-root GPIO access)
sudo groupadd -f -r gpio
sudo usermod -a -G gpio $USER
sudo cp /opt/nvidia/jetson-gpio/etc/99-gpio.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger
# Log out and back in for group changes to take effect
```

## 9. Edit dataset.yaml

Open `data/dataset.yaml` and update the `path` field to your actual home directory:

```yaml
path: /home/YOUR_USERNAME/chick_defect_project/data
```

---

## 10. Prepare Dataset

Place your annotated images (YOLO `.txt` format) in the class subfolders under `data/images/`.
Then run:

```bash
conda activate chick_env
python scripts/prepare_dataset.py
```

This will:
- Split images: 70% train / 20% val / 10% test (stratified)
- Augment the minority Normal class with horizontal flips
- Write the final `data/dataset.yaml`

---

## 11. Train the Model

```bash
conda activate chick_env
python scripts/train.py --model yolo11n.pt --epochs 200 --batch 8
```

Training logs and weights are saved to `models/<run_name>/`.
The best weights are at `models/<run_name>/weights/best.pt`.

**Tips for Jetson Orin Nano:**
- Use `--batch 8` to start; reduce to 4 if you get out-of-memory errors.
- `--amp True` is enabled by default (mixed precision = faster + less VRAM).
- Training ~200 epochs on 70 images takes approximately 30вЂ“60 minutes on the Orin Nano GPU.

---

## 12. Evaluate the Model

```bash
conda activate chick_env
python scripts/evaluate.py --model models/<run_name>/weights/best.pt
```

Output:
- Console: mAP@50, Precision, Recall, F1, binary confusion matrix
- `logs/eval_report_<timestamp>.csv`
- `logs/confusion_matrix_<timestamp>.png`
- `logs/per_class_metrics_<timestamp>.png`

## 13. Run the System

```bash
conda activate chick_env

# Simulation mode (no GPIO/servo, safe for development)

# Headless mode (console only, no display needed)
```

## 14. GPIO Wiring Reference

| Component        | Jetson Pin | Notes                                  |
|-----------------|-----------|----------------------------------------|
| Servo Signal     | Pin 32    | PWM signal (3.3V logic is sufficient)  |
| Servo VCC        | External  | Use 5V external supply, NOT Jetson 3.3V|
| Servo GND        | Pin 6/9   | Common GND with external supply        |
| LED Strip        | External  | 12V supply, always-on                  |
| Camera Left      | USB 3.0   | /dev/video0                            |
| Camera Right     | USB 3.0   | /dev/video2                            |

> вљ пёЏ **Never power the servo motor from the Jetson's 3.3V or 5V pin.**
> The servo draws too much current and can cause brownouts or damage the board.

## 15. Troubleshooting

| Problem | Solution |
|---------|----------|
| `CUDA not available` | Verify JetPack PyTorch wheel is installed (Step 5) |
| Camera not found | Run `v4l2-ctl --list-devices` and adjust `--cam0` / `--cam1` |
| GPIO permission denied | Complete Step 7 and re-login |
| `ultralytics not found` | `pip install ultralytics --break-system-packages` |
| Out of memory during training | Reduce `--batch` to 4 or use `yolo11n.pt` (nano) |
| PyQt5 display error | Ensure `DISPLAY=:0` is set; or use `--no_gui` mode |

## 16. Auto-Start on Boot (Optional)

To launch the system automatically when the Jetson boots:

```bash
# Create a systemd service
sudo nano /etc/systemd/system/chick-detection.service
```

Paste:
```ini
[Unit]
Description=Cobb 500 Chick Defect Detection System
After=network.target graphical-session.target

[Service]
User=YOUR_USERNAME
WorkingDirectory=/home/YOUR_USERNAME/chick_defect_project
Environment=DISPLAY=:0
ExecStart=/home/YOUR_USERNAME/miniconda3/envs/chick_env/bin/python main.py \
          --model models/best.pt
Restart=on-failure

[Install]
WantedBy=graphical-session.target
```

```bash
sudo systemctl enable chick-detection
sudo systemctl start  chick-detection
```
